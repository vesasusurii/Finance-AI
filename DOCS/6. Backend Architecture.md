## 2. Layer Descriptions

| Layer | What It Is | What It Does NOT Do |
|---|---|---|
| **main.py** | Creates app, lifespan (engine, httpx, storage), registers middleware and routers | No business logic |
| **config.py** | Env vars: `DATABASE_URL`, `OPENAI_API_KEY`, `STORAGE_PATH`, auth secrets | No logic |
| **core/exceptions.py** | Named errors: `ExtractionError`, `ExcelParseError`, `MatchError`, `ExportError` | Does not catch — only defines |
| **middleware/** | Logging, CORS, auth on protected routes | No business rules |
| **models/** | SQLAlchemy models mirroring Postgres tables | Never used outside repositories |
| **schemas/** | Request/response shapes | No DB access |
| **db/** | Async engine + session factory + Alembic | No queries |
| **repositories/** | CRUD and queries per aggregate | No matching rules, no AI calls |
| **services/** | Extraction, matching, Excel, review decisions | No HTTP status codes |
| **utils/** | Pure functions: parser, normalization, file paths | No HTTP, no sessions |
| **controllers/** | Validate input, call services, map exceptions to HTTP | No SQL, no OpenAI directly |
| **routers/** | URL → controller method | Zero logic |

## 3. Communication Rules

### Call Direction

```
Router → Controller → Service → Repository
                              ↘ utils/ (pure helpers, called from services)
```

No layer calls upward. Services may call `utils/` but not routers.

### What Crosses Boundaries

- **Schemas** cross every boundary.
- **Models** stay inside repositories (map to schemas before return).
- **Exceptions** rise to controllers; `request_handler` catches unhandled `AppError`.
- **AsyncSession** injected into repositories only via `dependencies.py`.

## 5. Full Traced Example — Bank Comment Match

Finance has already uploaded invoice `26063` (AI extracted). Bank statement row:

> **Komenti / Comment:** `Payment for invoice 26063`  
> **Data / Date:** `25.02.2026`

### Middleware

`request_handler` assigns `req-a91c`, logs `POST /api/reconciliation/run`. `auth` verifies Finance role.

### Router — reconciliation_router.py

Maps to `ReconciliationController.run_reconciliation`.

### Controller

Validates optional filters (statement id, date range). Calls `matching_service.run()`. Returns `ReconciliationResultResponse` with counts: matched, unmatched, review_created.

### Service — matching_service.py

1. Loads `bank_transactions` where `reconciliation_status = pending` (or all for statement id).
2. For each transaction:
   - `numbers = extract_invoice_numbers(transaction.comment)` from `utils/invoice_number_parser.py`
   - Normalizes via `utils/normalization.py`
   - Persists `detected_invoice_numbers` on the row
3. For each `invoice_number` in `numbers`:
   - `invoice = invoice_repo.find_by_normalized_number(normalized)` *(uses `invoice_number_normalized`)*
   - If found and `paid_at_date` is null (or overwrite policy allows):
     - Sets `invoice.paid_at_date = transaction.transaction_date`
     - Sets `invoice.match_status = Matched`
     - Creates `InvoicePaymentMatch` via `match_repo`
     - `audit_repo.log_payment_date_update(...)`
   - Else:
     - `review_repo.create_bank_match_failure(transaction, invoice_number)`

**Does not set `paid_by`.**

### Repository — invoice_repository.py

`update_paid_at_date(invoice_id, date)` — SQLAlchemy update on `Invoice` model. Returns `InvoiceResponse` schema.

### What Finance sees

Matching Results: Invoice 26063 — **Matched** — paid at (Date) = 25.02.2026. Export Excel includes the date; paid by still empty for manual entry.

### If extraction had failed earlier

Invoice `26063` never in DB → match step creates review task: *"Bank reference 26063 — no invoice in database"*.

## 6. Full Traced Example — Multi-Invoice Comment

**Comment:** `Payment for invoices 26063, 26060, 260658`  
**Date:** `25.02.2026`

`extract_invoice_numbers` → `["26063", "26060", "260658"]`

Loop: three lookups, three `paid_at_date` updates (same date), three `invoice_payment_matches` rows. If `260658` not in DB → review task for that number only; others still match.

## 7. Exception Flow

```
Repository  → RAISES (DB, not found)
Service     → RAISES or returns partial result with review flags
Controller  → CATCHES → HTTP 4xx/5xx + message
Middleware  → CATCHES unhandled AppError → 500 JSON
```

| Exception | Typical HTTP | When |
|---|---|---|
| `ExtractionError` | 422 | AI/OCR failed on a file |
| `ExcelParseError` | 400 | Bank Excel missing required headers |
| `MatchError` | 500 | Reconciliation internal failure |
| `ExportError` | 500 | Excel generation failed |

Services return business outcomes (e.g. match summary) rather than throwing when a single row fails — failed rows go to `review_tasks`.

## 8. Implementation Patterns

### Stack Rules

| Concern | Implementation |
|---|---|
| ORM | SQLAlchemy 2.0 async |
| DB driver | `asyncpg` via SQLAlchemy |
| Migrations | Alembic autogenerate from models |
| HTTP client | `httpx.AsyncClient` (OpenAI, optional Document AI) |
| Excel | `openpyxl` |
| Schemas | `pydantic.BaseModel` |
| Config | `pydantic-settings` |
| DI | FastAPI `Depends()` in `dependencies.py` |
| File uploads | `UploadFile` + `file_storage` |
| Auth | JWT or session middleware — **required** for production |

### Router pattern

```python
@router.post("/upload")
async def upload_invoices(
    files: list[UploadFile],
    ctrl: InvoiceController = Depends(get_invoice_controller),
    user: UserContext = Depends(get_current_user),
):
    return await ctrl.upload(files, user)
```

### Service pattern — matching (conceptual)

```python
class MatchingService:
    async def run(self, bank_statement_id: int | None = None) -> ReconciliationSummary:
        transactions = await self.bank_txn_repo.list_pending(bank_statement_id)
        summary = ReconciliationSummary()
        for txn in transactions:
            candidates = extract_invoice_numbers(txn.comment)
            await self.bank_txn_repo.save_detected_numbers(txn.id, candidates)
            for raw in candidates:
                key = normalize_invoice_number(raw)
                invoice = await self.invoice_repo.find_by_number(key)
                if invoice:
                    await self._apply_match(invoice, txn, key, summary)
                else:
                    await self.review_repo.create_bank_unmatched(txn, key, summary)
        return summary
```

## 9. Security (backend responsibilities)

> Full policy: [11. Security & Compliance.md](./11. Security & Compliance.md)

| Requirement | Implementation |
|---|---|
| No bank API | Only `bank_statement_service` reads uploaded Excel |
| Secure file storage | `file_storage` outside public web root; signed URLs if needed |
| Authentication | `middleware/auth.py` on all `/api/*` except health |
| RBAC | Finance vs admin roles on destructive/export routes |
| Audit | `audit_repository` on extraction, edits, match, paid date change |
| Original files retained | `uploaded_files.storage_path` never deleted on soft invoice delete |
