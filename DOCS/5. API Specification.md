# API Specification — Borek Finance Invoice Automation

> **Base URL:** `/api`  
> **Format:** All request and response bodies are JSON unless the endpoint handles file upload (multipart) or file download (binary).  
> **Auth:** Every endpoint except `POST /api/auth/login` and `GET /api/health` requires a valid session. Unauthenticated requests return `401`.

---

## Conventions

### Authentication

```
POST /api/auth/login → JWT stored in httpOnly cookie (Set-Cookie)
All subsequent requests send the cookie automatically
401 Unauthorized → redirect to /login
```

| Cookie | Value |
|---|---|
| `access_token` | Signed JWT — contains `user_id`, `role`, `exp` |
| `HttpOnly` | Yes — not readable by JavaScript |
| `SameSite` | `Lax` |

### Standard error shape

Every non-2xx response returns:

```json
{
  "error": "short_error_code",
  "message": "Human-readable description of what went wrong."
}
```

### Pagination

List endpoints support optional query params:

| Param | Type | Default | Description |
|---|---|---|---|
| `page` | integer | `1` | Page number (1-indexed) |
| `limit` | integer | `50` | Results per page (max 200) |

Paginated responses include:

```json
{
  "items": [...],
  "total": 240,
  "page": 1,
  "limit": 50
}
```

### Date format

All dates are `YYYY-MM-DD` strings. All timestamps are ISO 8601 UTC (`2026-05-20T10:00:00Z`).

### Status enums

| Field | Values |
|---|---|
| `processing_status` | `pending`, `processing`, `processed`, `failed` |
| `review_status` | `pending`, `approved`, `needs_review` |
| `match_status` | `unmatched`, `matched`, `needs_review` |
| `reconciliation_status` | `pending`, `matched`, `partial`, `needs_review` |
| `match_record_status` | `matched`, `approved`, `rejected` |

## Auth

### `POST /api/auth/login`

Authenticate a Finance or Admin user.

**Request body:**

```json
{
  "email": "finance@borek.com",
  "password": "secret"
}
```

**Response `200`:**

```json
{
  "user_id": 1,
  "email": "finance@borek.com",
  "role": "finance"
}
```

Sets `access_token` httpOnly cookie. No token in body.

**Errors:**

| Status | Code | When |
|---|---|---|
| `401` | `invalid_credentials` | Wrong email or password |
| `403` | `account_disabled` | `is_active = false` |

---

### `POST /api/auth/logout`

Clears the `access_token` cookie.

**Response `200`:**

```json
{ "message": "Logged out." }
```

---

### `GET /api/auth/me`

Returns the currently authenticated user.

**Response `200`:**

```json
{
  "user_id": 1,
  "email": "finance@borek.com",
  "role": "finance"
}
```

## Invoices

### `POST /api/invoices/upload`

Upload one or many invoice files for AI extraction. Files are processed independently.

**Request:** `multipart/form-data`

| Field | Type | Required | Notes |
|---|---|---|---|
| `files` | `File[]` | Yes | PDF, JPG, JPEG, PNG, scanned PDF. Multiple allowed. |

**Response `202`:**

```json
{
  "uploaded": 3,
  "items": [
    {
      "upload_id": 101,
      "original_filename": "adia_invoice_may.pdf",
      "processing_status": "pending"
    },
    {
      "upload_id": 102,
      "original_filename": "invoice_26060.jpg",
      "processing_status": "pending"
    }
  ]
}
```

`202 Accepted` — files are queued for extraction. Poll `GET /api/invoices/{id}` or the invoice list for status updates.

**Errors:**

| Status | Code | When |
|---|---|---|
| `400` | `no_files` | No files attached to the request |
| `415` | `unsupported_file_type` | File type not in allowed list |
| `413` | `file_too_large` | Single file exceeds size limit |

```json
  "client_employee_related": "",
  "paid_by": "Finance Team",
```

|---|---|---|
| `404` | `invoice_not_found` | |

## Bank Statements

### `POST /api/bank-statements/upload`

Upload a Bank Statement Excel file. Backend reads and parses rows immediately.

**Request:** `multipart/form-data`

| Field | Type | Required | Notes |
|---|---|---|---|
| `file` | `File` | Yes | `.xlsx` or `.xls` |

**Response `200`:**

```json
{
  "bank_statement_id": 5,
  "row_count": 48,
  "processing_status": "processed",
  "preview": [
    {
      "transaction_date": "2026-02-25",
      "debited_amount": 772.39,
      "credited_amount": null,
      "transaction_type": "Transfer",
      "comment": "Payment for invoices 26063, 26060, 260658",
      "detected_invoice_numbers": ["26063", "26060", "260658"]
    }
  ]
}
```

`preview` contains up to the first 10 rows so Finance can verify the parse before proceeding.

**Errors:**

| Status | Code | When |
|---|---|---|
| `400` | `missing_required_columns` | Excel missing `Komenti / Comment` or `Data / Date` column |
| `400` | `empty_file` | No data rows found |
| `415` | `unsupported_file_type` | Not an Excel file |

## Reconciliation

### `POST /api/reconciliation/run`

Run the matching engine. For each bank transaction, detected invoice numbers are normalized and compared against the `invoices` table. Matches set `paid_at_date`; failures create review tasks.

**Request body** (all optional):

```json
{
  "bank_statement_id": 5
}
```

Omitting `bank_statement_id` runs matching across all statements with pending transactions.

**Response `200`:**

```json
{
  "matched": 35,
  "unmatched_invoices": 12,
  "unmatched_transactions": 5,
  "review_tasks_created": 7,
  "run_at": "2026-05-20T10:15:00Z"
}
```

---

### `GET /api/reconciliation/results`

Summary of the most recent reconciliation run, with detail rows.

**Query params:**

| Param | Type | Description |
|---|---|---|
| `status` | string | `matched`, `unmatched`, `needs_review` |
| `bank_statement_id` | integer | Scope to one statement |
| `page` | integer | |
| `limit` | integer | |

**Response `200`:**

```json
{
  "items": [
    {
      "id": 1,
      "invoice_id": 10,
      "bank_transaction_id": 1,
      "invoice_number": "26063",
      "match_type": "invoice_number",
      "match_confidence": 1.0,
      "status": "matched",
      "paid_at_date": "2026-02-25",
      "created_at": "2026-05-20T10:15:00Z"
    }
  ],
  "total": 35,
  "page": 1,
  "limit": 50
}
```

---

### `POST /api/reconciliation/approve-match`

Finance confirms a suggested match. Sets `invoice_payment_matches.status` to `approved`.

**Request body:**

```json
{
  "match_id": 1
}
```

**Response `200`:**

```json
{
  "match_id": 1,
  "status": "approved"
}
```

**Errors:**

| Status | Code | When |
|---|---|---|
| `404` | `match_not_found` | |
| `409` | `match_already_resolved` | Already approved or rejected |

---

### `POST /api/reconciliation/reject-match`

Finance rejects a suggested match. Removes `paid_at_date` from the invoice and returns `match_status` to `needs_review`.

**Request body:**

```json
{
  "match_id": 1,
  "reason": "Wrong invoice number in comment"
}
```

`reason` is optional but encouraged for the audit log.

**Response `200`:**

```json
{
  "match_id": 1,
  "status": "rejected"
}
```

| Param | Type | Description |
|---|---|---|
| `task_type` | string | `extraction` or `bank_match` |
| `status` | string | Default `open` |
| `page` | integer | |

### `POST /api/review/{id}/approve`

**Request body:** *(empty)*

**Response `200`:**

```json
  "resolved_at": "2026-05-20T11:00:00Z"

```

## Export

### `GET /api/export/purchase-invoices-excel`

Download the Purchase Invoices Database as an `.xlsx` file with the official Finance column structure.

**Query params (all optional):**

| Param | Type | Example | Description |
|---|---|---|---|
| `invoice_date_from` | date | `2026-01-01` | Filter by invoice date |
| `invoice_date_to` | date | `2026-05-31` | |
| `match_status` | string | `matched` | Only export matched/unmatched/all |
| `review_status` | string | `approved` | |
| `category` | string | `Utilities` | |

**Response `200`:**

```
Content-Type: application/vnd.openxmlformats-officedocument.spreadsheetml.sheet
Content-Disposition: attachment; filename="purchase_invoices_export_2026-05-20.xlsx"
```

Binary `.xlsx` file. Column order matches the official Finance file exactly:

1. Invoice Date  
2. Name of Company  
3. Adress of Company  
4. Invoice Number  
5. Amount  
6. Account Details  
7. Internal Note/Description  
8. Client / Employee Related  
9. paid at (Date)  
10. paid by  
11. Fixed/Not fixed  
12. Category  

**Errors:**

| Status | Code | When |
|---|---|---|
| `500` | `export_failed` | Excel generation error |
