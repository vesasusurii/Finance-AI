# Security & Compliance

> **Principle:** This system processes **financial documents and payment data**. It must be **internal-only**, **authenticated**, **audited**, and **never connected to live bank APIs**. When in doubt, **deny access** and **log the attempt** — do not expose files or guess financial fields.

**Source:** Project specification (Section 16 — Security Requirements), aligned with [2. Tech Stack.md](./2. Tech Stack.md), [6. Backend Architecture 1.md](./6. Backend Architecture 1.md), [7. DB Schema 1.md](./7. DB Schema 1.md).

**Related docs:** [3. Technical Workflow 1.md](./3. Technical Workflow 1.md) · [8. OCR-Technology.md](./8. OCR-Technology.md) · [10. Excel-File-Formats.md](./10. Excel-File-Formats.md) · [9. Matching-Normalization.md](./9. Matching-Normalization.md)

| In scope | Out of scope (MVP) |
|---|---|

## Security Rules (from project specification)

These rules are **mandatory** for MVP and production.

| # | Rule | Implementation |
|---|---|---|
| 1 | **No direct bank connection** | Bank data only from uploaded Excel — [10. Excel-File-Formats.md](./10. Excel-File-Formats.md) |
| 2 | **No public invoice access** | Files served only through authenticated API; no static public URLs |
| 3 | **Store files securely** | Private storage path / object store; outside web root |
| 4 | **Authentication required** | All `/api/*` except health check |
| 5 | **Role-based access** | `finance` vs `admin` — see [RBAC](#role-based-access-rbac) |
| 6 | **Log all edits** | `audit_logs` on invoice field changes |
| 7 | **Log all AI extractions** | `audit_logs` + `extraction_confidence` on `invoices` |
| 8 | **Log all payment date updates** | `audit_logs` when `paid_at_date` changes via matching |
| 9 | **Keep original files for audit** | Do not delete `uploaded_files` blob when row soft-deleted |
| 10 | **Allow manual correction** | Finance can edit fields; every change audited |

**Business logic (also security-related):**

- Never auto-fill `paid_by` — [3. Technical Workflow 1.md](./3. Technical Workflow 1.md)
- Never guess invoice numbers or amounts when confidence is low — [9. Matching-Normalization.md](./9. Matching-Normalization.md)
- `paid_at_date` only from bank matching, never from OCR — [8. OCR-Technology.md](./8. OCR-Technology.md)

## Threat model (simplified)

| Threat | Mitigation |
|---|---|
| Unauthorized access to invoices / bank data | Auth + RBAC + private file storage |
| Leaked API keys (OpenAI, DB) | Secrets in env / vault; never in frontend or git |
| Tampering with `paid_at_date` | Audit log + optional match approval gate |
| Malicious file upload | MIME/size checks; scan path; no execution of uploaded content |
| Data sent to wrong AI region | Configure OpenAI/Google region; DPAs with Finance |
| IDOR on invoice/file download | Authorize every `GET` by `user_id` + role |
| Log leakage (IBAN, full account numbers) | Redact in application logs; full values only in DB |

## Authentication

### Model (MVP)

| Item | Choice |
|---|---|
| Users | Finance team only — accounts created by admin |
| Method | Email + password (bcrypt/argon2 hash in `users.password_hash`) or SSO later |
| Session | JWT in `Authorization: Bearer` header **or** HTTP-only secure cookie |
| Token lifetime | Short access token (e.g. 8h) + refresh optional in Phase 2 |

### Public endpoints (no auth)

| Route | Purpose |
|---|---|
| `GET /health` | Load balancer / monitoring |
| `POST /api/auth/login` | Issue token |

### Protected endpoints

All other `/api/*` routes require valid authentication — enforced in `middleware/auth.py` ([6. Backend Architecture 1.md](./6. Backend Architecture 1.md)).

```python
# Conceptual — every protected route
async def get_current_user(authorization: str = Header(...)) -> User:
    user = verify_jwt(authorization)
    if not user or not user.is_active:
        raise HTTPException(401, "Unauthorized")
    return user
```

### Password policy (recommended)

| Rule | Value |
|---|---|
| Minimum length | 12 characters |
| Lockout | Optional after N failed logins |
| Storage | Never store plaintext passwords |

## Role-based access (RBAC)

| Role | Permissions |
|---|---|
| **`finance`** | Upload invoices & bank files; view/edit invoices; run reconciliation; approve review items; export Excel |
| **`admin`** | All `finance` permissions + manage users + view full audit logs |

| Action | `finance` | `admin` |
|---|---|---|
| `POST /api/invoices/upload` | Yes | Yes |
| `PUT /api/invoices/{id}` | Yes | Yes |
| `POST /api/bank-statements/upload` | Yes | Yes |
| `POST /api/reconciliation/run` | Yes | Yes |
| `GET /api/export/purchase-invoices-excel` | Yes | Yes |
| `DELETE /api/invoices/{id}` | Optional | Yes |
| User management | No | Yes |

Destructive or sensitive routes check role in controller or `dependencies.py`:

```python
def require_admin(user: User = Depends(get_current_user)) -> User:
    if user.role != "admin":
        raise HTTPException(403, "Admin only")
    return user
```

## File & document security

### Invoice files (PDF / images)

| Control | Detail |
|---|---|
| Storage | `uploaded_files.storage_path` — not under `public/` |
| Download | `GET /api/files/{id}` — auth required; verify user may access parent invoice |
| Upload validation | Allowlist MIME: `application/pdf`, `image/jpeg`, `image/png` |
| Max size | e.g. 15 MB PDF, 10 MB image — reject larger |
| Filename | Sanitize `original_filename`; store UUID on disk |
| Execution | Never run uploaded files as code |

### Bank statement Excel

| Control | Detail |
|---|---|
| Source | User upload only — no bank API ([10. Excel-File-Formats.md](./10. Excel-File-Formats.md)) |
| Parsing | Server-side `openpyxl` / `xlrd` — no macros execution |
| Retention | Keep original `.xls` / `.xlsx` linked to `bank_statement_id` |
| Scope | Import transaction rows only (e.g. **13–96**); skip summary/footer rows |

### Excel export (download)

| Control | Detail |
|---|---|
| Auth | `GET /api/export/purchase-invoices-excel` requires login |
| Audit | Log `export_run`: `user_id`, timestamp, row count, format (`xlsx`/`xls`) |
| Content | Same access as invoice table — no export of other tenants’ data (single-org MVP) |

## Third-party AI & data processing

Invoice OCR sends document content to external APIs — treat as **data processing**.

| Provider | Data sent | When | Fallback |
|---|---|---|---|
| **OpenAI Vision** (primary) | Invoice page images / text | Every extraction | Google Document AI |
| **Google Document AI** (optional) | Invoice bytes | When OpenAI unavailable | — |
| **Bank Excel** | — | **Never** sent to OCR vendors | — |

### Controls

| Control | Detail |
|---|---|
| API keys | `OPENAI_API_KEY`, `GOOGLE_APPLICATION_CREDENTIALS` in server env only |
| Frontend | Never embed keys; backend proxies all AI calls |
| Minimization | Send invoice pages only; not full disk directories |
| Logging | Log `provider`, `model`, `file_id`, `confidence` — not full document text in logs |
| Failover audit | `ocr_failover` event when switching to Google — [8. OCR-Technology.md](./8. OCR-Technology.md) |
| Contracts | Finance must approve OpenAI / Google DPAs before production |

Bank statement rows stay on your infrastructure — parsed locally from Excel.

## Audit logging

Immutable-style append log in `audit_logs` ([7. DB Schema 1.md](./7. DB Schema 1.md)).

### Events to log (minimum)

| `action` | `entity_type` | When |
|---|---|---|
| `invoice_extracted` | `invoice` | OCR completes (success or needs_review) |
| `ocr_failed` | `uploaded_file` | Extraction failure |
| `invoice_updated` | `invoice` | Manual edit in UI |
| `invoice_approved` | `invoice` | Review queue approve |
| `bank_statement_imported` | `bank_statement` | Excel ingest completes |
| `reconciliation_run` | `bank_statement` | Matching job starts/ends |
| `payment_date_set` | `invoice` | `paid_at_date` written |
| `match_approved` | `match` | User confirms match |
| `match_rejected` | `match` | User rejects match |
| `export_run` | `export` | Excel download |
| `login_success` / `login_failed` | `user` | Auth attempts (no password in log) |

### Payload shape

```json
{
  "user_id": 3,
  "action": "payment_date_set",
  "entity_type": "invoice",
  "entity_id": 42,
  "before": { "paid_at_date": null },
  "after": { "paid_at_date": "2026-02-25", "bank_transaction_id": 18 },
  "created_at": "2026-05-21T14:00:00Z"
}
```

### What not to log

| Do not log | Why |
|---|---|
| Full IBAN / card numbers in plain text | PCI/PII risk — redact to last 4 in debug logs |
| OpenAI API keys | Secret leakage |
| Passwords | Never |
| Entire invoice OCR JSON in every info log | Use `audit_logs` + DB row; hash optional |

## Application & API security

| Control | MVP requirement |
|---|---|
| HTTPS | TLS in production — no plain HTTP except local dev |
| CORS | Allowlist frontend origin only — not `*` in production |
| CSRF | If cookie auth — use SameSite + CSRF token; if JWT header only — lower risk |
| Rate limiting | Login + upload endpoints (optional Phase 1b) |
| SQL injection | SQLAlchemy parameterized queries only |
| Input validation | Pydantic schemas on all request bodies |
| Error messages | Generic 500 to client; details in server logs only |
| Dependency updates | Pin versions; review `pip audit` / `npm audit` before release |

### IDOR prevention checklist

| Endpoint | Check |
|---|---|
| `GET /api/invoices/{id}` | User authenticated (org-scoped when multi-tenant later) |
| `GET /api/files/{id}` | File belongs to invoice user can access |
| `POST /api/reconciliation/approve-match` | Match id exists and user has finance role |

## Data retention & deletion

| Data type | Retention (default — confirm with Finance) |
|---|---|
| Original invoice PDF/image | Keep while invoice exists + audit period (e.g. 7 years) |
| Original bank Excel | Keep with `bank_statement_id` |
| `audit_logs` | Append-only; no hard delete in app |
| PostgreSQL backups | Encrypted backups per host policy |
| Exported Excel | Client-side — Finance responsibility after download |

| Action | Behavior |
|---|---|
| Soft-delete invoice | Keep `uploaded_files` blob and `audit_logs` |
| Hard-delete user | Deactivate (`is_active=false`); do not cascade-delete financial rows |

## Human review & compliance with business rules

Security includes **preventing wrong automation**:

| Risk | Control |
|---|---|
| Wrong `paid_at_date` | Matching only on detected invoice numbers; review queue for ambiguity |
| Wrong OCR amount | Confidence tiers + manual review — [8. OCR-Technology.md](./8. OCR-Technology.md) |
| Over-trusted AI | `needs_review` flag; export filter optional for non-approved rows |
| Override without trace | All manual edits → `audit_logs` |

Finance must be able to **approve**, **reject**, and **correct** before data is treated as final.

## Developer rules (interns)

| Do | Don't |
|---|---|
| Run auth middleware on new routes | Add public routes without review |
| Call `audit_repository.log_*` on money field changes | Update `paid_at_date` without audit |
