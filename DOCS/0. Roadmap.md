## Before Writing Any Code — Rules That Never Change

These rules apply in every phase. Violating them corrupts financial data.

| Rule | Source |
|---|---|
| `paid_at_date` is written **only** by `matching_service` after a bank statement match | Doc 1, Doc 3 §4 |
| `paid_by` is **never** written by the system — Finance fills it manually | Doc 1, Doc 3 §4.6 |
| The AI must **never guess** invoice numbers when confidence is low | Doc 1 core principle |
| All status enum values are **lowercase**: `pending`, `approved`, `needs_review`, `matched`, `unmatched` | Doc 5 conventions |
| No direct bank API connection — only uploaded Excel files | Doc 1, Doc 3 §3 |
| All financial edits are logged in `audit_logs` | Doc 7, Doc 6 §9 |
| User deactivation = `is_active = false` — do not delete users who have financial history | Doc 7 users table |

## Build Order Principles

### Backend — always in this sequence per feature

```
Schema (schemas/) → Model (models/) → Alembic migration → Repository (repositories/)
  → Service (services/) → Controller (api/controllers/) → Router (api/routers/)
  → Wire in dependencies.py
```

Source: Doc 6 §8 "New feature checklist"

### Frontend — always in this sequence per feature

```
types/ → api/ → hooks/ → pages/ → add route in App.tsx
```

Source: Doc 4 §8 "New page checklist"

### Layer boundaries — never cross these

| Layer | Calls | Never calls |
|---|---|---|
| Router | Controller only | Nothing else |
| Controller | Service only | Repositories, OpenAI, DB directly |
| Service | Repositories, utils/ | Routers, controllers, HTTP status codes |
| Repository | DB session (AsyncSession) | Services, other repositories |
| utils/ | Pure Python only | Any layer |

Source: Doc 6 §3 communication rules

### 0.4 — Create middleware

Files: `backend/middleware/__init__.py`, `backend/middleware/cors.py`, `backend/middleware/request_handler.py`, `backend/middleware/auth.py`

**cors.py:** Read `settings.cors_origins`, split on comma, configure `CORSMiddleware`. Set `allow_credentials=True`.

**request_handler.py:** For every request:
- Assign correlation ID (`X-Correlation-ID` header or generate UUID)
- Log `{method} {path} — started` with correlation ID
- On response: log `{method} {path} — {status} — {elapsed_ms}ms`
- Catch any unhandled `AppError` subclass → return `{"error": "internal_error", "message": str(e)}` with HTTP 500
- Standard error response shape (Doc 5 conventions): `{"error": "...", "message": "..."}`

**auth.py:** Middleware or FastAPI dependency (dependency preferred — cleaner DI).
- Exempt paths: `/api/auth/login`, `/api/health`, `/api/ready`
- All other `/api/*` paths: extract `access_token` httpOnly cookie → decode JWT using `settings.jwt_secret` → attach `UserContext` to request state
- Return `{"error": "unauthorized", "message": "Authentication required."}` HTTP 401 if token missing or invalid

### 1.4 — Backend services (Phase 1)

**ai_validation_service.py** — confidence tier logic (Doc 2 §1, Doc 3 Step 2):

```python
def determine_review_status(result: ExtractionResult) -> str:
    if result.confidence_score >= 0.90 and not result.needs_review:
        return "pending"   # auto-saved, no review needed
    if result.confidence_score >= 0.70:
        return "needs_review"
    return "needs_review"  # below 0.70 — always flag

def validate_required_fields(result: ExtractionResult) -> list[str]:
    """Return list of missing critical fields: invoice_number, amount, name_of_company, invoice_date."""
```

**invoice_extraction_service.py** — Doc 3 Step 2 full flow:

1. Receive `UploadFile` and `UserContext`
2. Save file via `file_storage.save_upload()` → get `storage_path`
3. Create `uploaded_files` row via `upload_repository.create()` with `processing_status='processing'`
4. Read file bytes, base64-encode for OpenAI Vision
5. Call `openai_client.chat.completions.create()` with model from `settings.openai_model`:
   - System prompt: instruct to return only valid JSON matching `ExtractionResult` schema — no prose
   - User message: base64 image or extracted text from PDF
6. Parse JSON response → `ExtractionResult`
7. Call `ai_validation_service.determine_review_status(result)` → `review_status`
8. Call `invoice_repository.create(result, source_file_id, review_status)`
9. Call `audit_repository.log(user_id, 'extraction', 'invoice', invoice.id, None, result.dict())`
10. Call `upload_repository.update_status(upload_id, 'processed')`
11. On any OpenAI/parse failure: `upload_repository.update_status(upload_id, 'failed')`, raise `ExtractionError`

```python
def write_purchase_invoices_workbook(invoices: list[InvoiceResponse]) -> bytes:
```

Column order must match exactly (Doc 7):

### 1.5 — Backend auth controller and router

**auth_controller.py:**
- `login(request: LoginRequest) -> Response` — verify bcrypt hash, sign JWT with `user_id`, `email`, `role`, `exp`; set `access_token` httpOnly cookie (`secure=settings.cookie_secure`, `samesite=settings.cookie_samesite`); return `LoginResponse`

```python

POST /api/auth/logout  → auth_controller.logout
GET  /api/auth/me      → auth_controller.me
```

All three paths must be in `auth.py` middleware exempt list — login and logout need no auth; `me` reads the decoded token already in request state.

### 3.4 — Backend service: matching_service.py

This is the core business logic of the entire project. Implement exactly per Doc 3 §4, Doc 6 §5–6.

```python
class MatchingService:
    async def run(self, bank_statement_id: int | None = None) -> ReconciliationSummary:
        summary = ReconciliationSummary(matched=0, unmatched_invoices=0,
                                        unmatched_transactions=0, review_tasks_created=0,
                                        run_at=datetime.utcnow().isoformat())

        transactions = await self.bank_txn_repo.list_pending(bank_statement_id)

        for txn in transactions:
            candidates = extract_invoice_numbers(txn.comment or "")
            await self.bank_txn_repo.save_detected_numbers(txn.id, candidates)

            if not candidates:
                # No numbers detectable — send to review
                await self.review_repo.create_bank_unmatched(
                    txn.id, "", "no_invoice_numbers_detected")
                summary.review_tasks_created += 1
                summary.unmatched_transactions += 1
                continue

            matched_any = False
            for raw in candidates:
                key = normalize_invoice_number(raw)
                invoice = await self.invoice_repo.find_by_number(key)

                if invoice:
                    # MATCH FOUND
                    await self.invoice_repo.update_paid_at_date(invoice.id, txn.transaction_date)
                    await self.invoice_repo.update_match_status(invoice.id, "matched")
                    await self.match_repo.create(
                        invoice_id=invoice.id,
                        txn_id=txn.id,
                        invoice_number=key,
                        match_type="invoice_number",
                        match_confidence=1.0,
                        paid_at_date=txn.transaction_date,
                    )
                    await self.audit_repo.log(
                        None, "payment_date_set", "invoice", invoice.id,
                        {"paid_at_date": None}, {"paid_at_date": str(txn.transaction_date)}
                    )
                    summary.matched += 1
                    matched_any = True
                else:
                    # Number detected but no invoice in DB
                    await self.review_repo.create_bank_unmatched(
                        txn.id, key, "no_invoice_in_db")
                    summary.review_tasks_created += 1

            if not matched_any:
                summary.unmatched_transactions += 1

        return summary
```

**Critical rules enforced by this service:**
- `paid_by` is **never set** here or anywhere in the codebase
- Multi-invoice comment: the `for raw in candidates` loop handles all detected numbers; each matched invoice gets the **same** `transaction_date` (Doc 3 §4.3)
- MVP: only exact normalized match; no amount-combination guessing (Doc 3 §4.4)
- One comment with 3 numbers, 2 in DB → 2 matched, 1 review task created

class ReviewService:
    async def list_open(self, task_type: str | None, page: int, limit: int):
        return await self.review_repo.list_open(task_type, page, limit)

    async def approve(self, task_id: int, user: UserContext):
        task = await self.review_repo.get(task_id)
        if task.task_type == "extraction":
            await self.invoice_repo.approve(task.invoice_id)
        elif task.task_type == "bank_match":
