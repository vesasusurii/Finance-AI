# Project Summary — Borek Finance Invoice Automation

> **Core principle:** AI can extract and suggest. The system auto-fills only on a clear match. Finance must review unclear cases. Never guess financial data when confidence is low.

---

## What This Project Is

Borek Finance currently manages purchase invoices manually in an Excel file called the **Purchase Invoices Database**. Matching those invoices to bank payments is done by hand — finding invoice numbers inside bank statement comments and filling in the paid date row by row.

This project builds an **AI-powered internal tool** that automates the repetitive parts while keeping Finance in control of all financial decisions.

---

## The Problem Being Solved

| Current manual process | Pain point |
|---|---|
| Reading each invoice file and typing fields into Excel | Time-consuming; error-prone |
| Opening the bank statement and scanning the Comment column for invoice numbers | Tedious; easy to miss |
| Manually copying the payment date into the right invoice row | Critical — wrong date = wrong accounts |
| Tracking which invoices have no bank match yet | No visibility; requires cross-checking two files |

---

## What the System Does

```
Finance uploads invoice files (PDF, photo, scan)
    ↓
AI reads and extracts structured invoice fields
    ↓
Data saved into Purchase Invoices Database
    ↓
Finance uploads Bank Statement Excel
    ↓
System detects invoice numbers from Komenti / Comment column
    ↓
System matches invoice numbers → fills paid at (Date)
    ↓
Unmatched or uncertain cases flagged for manual review
    ↓
Finance exports updated Purchase Invoices Database Excel
```

---

## Users

| User | Role |
|---|---|
| **Finance team** | Upload invoices, upload bank statements, review uncertain cases, approve matches, export Excel, manually fill `paid by` |
| **Admin** | Manage Finance user accounts, access all data |

No public access. No bank API connection. No external users.

---

## Main Files Used

### Purchase Invoices Database (main output)

The existing Excel file Finance uses today. The system reads from and writes to a database; the Excel is exported on demand with the same column structure.

| Column | Filled by |
|---|---|
| Invoice Date | AI |
| Name of Company | AI |
| Adress of Company | AI |
| Invoice Number | AI |
| Amount | AI |
| Account Details | AI |
| Internal Note/Description | AI |
| Client / Employee Related | AI suggestion or manual |
| **paid at (Date)** | **System — bank statement match only** |
| **paid by** | **Manual by Finance — never automated** |
| Fixed/Not fixed | Manual or dropdown |
| Category | AI suggestion + manual correction |

### Bank Statement Excel (upload only — no bank API)

| Column | Purpose |
|---|---|
| Data / Date | Payment date → written to `paid at (Date)` on match |
| Vlera e debituar / Debited amount | Amount paid out |
| Vlera e kredituar / Credited amount | Amount received |
| Tipi i veprimit / Type of transaction | Transaction type |
| **Komenti / Comment** | **Invoice number detection — most critical column** |

---

## Core Matching Rule

```
Bank Statement Komenti / Comment
  → detect invoice number(s)
  → find same Invoice Number in Purchase Invoices Database
  → copy Bank Statement Data / Date
  → write to paid at (Date)
```

This is the single most important function of the system. Everything else supports it.

---

## What the System Does NOT Do

| Out of scope | Why |
|---|---|
| Connect directly to any bank | Security — uploaded files only |
| Auto-fill `paid by` | Finance policy — always manual |
| Guess invoice numbers when comment is unclear | Financial data integrity rule |
| Replace Finance review for uncertain extractions | AI confidence thresholds; human approval required |
| Manage supplier relationships or contracts | Outside project scope |
| Send emails or notifications | Not in MVP |

## MVP Phases

| Phase | Deliverable |
|---|---|
| **1** | Upload invoice files → AI extract → Purchase Invoices Table → Export Excel |
| **2** | Upload Bank Statement Excel → read rows → detect invoice numbers from Comment |
| **3** | Match detected numbers → fill `paid at (Date)` → Needs Review queue |
| **4** | Manual Review Queue — Finance approves/edits extractions and matches |
| **5** | Reports — monthly totals, paid/unpaid breakdown, category summary |

---

## Definition of Done (MVP Complete)

- [ ] Finance uploads invoice files; AI extracts data into Purchase Invoices Table
- [ ] Finance uploads Bank Statement Excel; system reads `Komenti / Comment`
- [ ] System detects and normalizes invoice numbers from comments
- [ ] System matches invoice numbers and fills **paid at (Date)** automatically
- [ ] **paid by** remains empty for manual entry
- [ ] Unmatched and low-confidence rows appear in **Needs Review**
- [ ] Finance exports updated Purchase Invoices Excel with the same column structure as today
- [ ] All edits, extractions, and payment date updates are logged in the audit trail
